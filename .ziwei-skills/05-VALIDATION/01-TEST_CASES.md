---

## 1. 驗證流程

每次修改紫微斗數相關代碼後，**必須**執行以下驗證：

```bash
# 1. OpenAPI 契約驗證
openapi-generator validate -i contracts/openapi/ziwei-zenith.yaml

# 2. Go 編譯
go build ./...

# 3. 前端編譯
npm run build --prefix web
```

## 8. 迴歸測試

每次修改後，必須測試以下場景：

### 場景 1：初次載入
1. 輸入生日：1972/6/8 丑時
2. 點擊「查看命盤」
3. 預期：命盤正確顯示

### 場景 2：大限切換
1. 在「流運直觀操作面板」選擇不同大限
2. 預期：三方四正正確更新

### 場景 3：流年切換
1. 改變日期到不同年份
2. 預期：流年四化正確更新

### 場景 4：命盤宮位點擊
1. 點擊命盤中的不同宮位
2. 預期：三方四正邊框變色 + 飛星四化顯示

## 9. 常見錯誤警示

### 錯誤 1：四化顯示不完整
- **現象**：只有部分四化顯示
- **可能原因**：計算邏輯有誤或數據傳遞問題

### 錯誤 2：三方四正不同步
- **現象**：切換大限但三方四正不更新
- **可能原因**：`currentTemporalData` 未正確更新

### 錯誤 3：飛星四化錯亂
- **現象**：顯示的飛星四化與當前層級不符
- **可能原因**：`focus.label` 判斷邏輯有誤
